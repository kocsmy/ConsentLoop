# Priority Issues (forward-looking backlog)

1. **[P1] Playground accent swatches export WCAG-failing primary buttons.** `#swatches` sets only `tokens.accent`/`switch-on`; `--cl-accent-fg` stays #fff. Measured: Sky 2.77:1, Green 3.30:1, Amber 2.15:1 (light), Indigo 2.82:1 (dark preset). A consent vendor's own tool produces non-compliant consent UI while docs claim WCAG-friendly contrast. Fix: compute accent-fg by luminance per accent + inline contrast warning. → /impeccable harden
2. **[P1] Widget mobile touch-target rule is dead code.** styles.ts:197 `@media(max-width:640px){.cl-btn{min-height:42px}}` loses to base `.cl-root .cl-btn` specificity (0,2,0 vs 0,1,0) — CDP-verified overridden; mobile buttons render 38px, banner link 19px, brand 17px. Fix: `.cl-root .cl-btn` in the media query. → /impeccable adapt
3. **[P1] Mobile: live preview unreachable while adjusting; /docs/configuration overflows 23px @390.** Controls-then-preview stacking (1,500–4,000px separation) kills the core feedback loop; `.cfg-grid` single-column `1fr` lacks `minmax(0,1fr)` (configuration.html:30) — fix verified live (413→390). Fix: sticky collapsed preview on mobile + minmax. → /impeccable adapt
4. **[P1] Export handoff is a false promise; zero persistence.** Export dialog links to configurator "for npm/React variants" but no state transfers; refresh wipes both tools; preset click destroys work without undo. Fix: serialize state to URL hash read by both tools, or render npm/React tabs in the export dialog. → /impeccable harden
5. **[P2] Contrast debt in chrome + widget.** `--fg3` #8a8a94 = 3.42:1 on white at 11–13px (footer, breadcrumbs, sidebar/table headers, playground h3 labels — site-wide); widget `--cl-switch-off` #d4d4d8 track vs white = 1.48:1 (WCAG 1.4.11 needs 3:1); dark-theme focus ring 3.28:1 borderline. Fix: darken fg3 to ≈#6d6d78, switch-off to ≈#9a9aa2. → /impeccable polish
